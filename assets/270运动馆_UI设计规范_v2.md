## 更新日志 v2.0

### 体系架构调整
- 移除底部 Tab Bar，移动端统一顶部汉堡菜单 + 侧边栏
- 移除管理后台仪表盘（独立项目）
- 饮食/营养页 → 合并为「健康生活」内容板块
- 个人中心 → 极简化为账户概览（顶部 Tab 切换，非后台式布局）
- 首页从 8 个板块压缩为 6 个，每屏接近全屏，强化叙事感
- 移除独立数据展示区，数据融入品牌故事区底部

### 色彩与视觉语言
- 橙色严格控制在 5% 以内，仅用于 CTA 按钮、Logo、关键数据强调
- 移除玫瑰金、鼠尾草绿等非品牌色，辅助色仅保留奶油白、暖沙色
- VIP/会员用深灰 + 橙色点缀 + 白色文字
- 输入框焦点、Tab 选中、标签、评分、筛选等全部去橙色化
- CTA 区块背景改深灰（Gray-900）+ 橙色按钮点缀
- 暖沙色仅小面积点缀，不大面积铺

### 视觉克制化
- 圆角整体收小：卡片 12px、按钮 8px、输入框 8px
- 阴影大幅减少：默认卡片无阴影，仅 1px 极细边框
- 阴影仅限真正悬浮的元素（下拉、弹窗、浮卡）
- 卡片悬停：极微缩放（scale 1.01）+ 边框加深，不上移不加阴影
- 品牌英文标题改用无衬线粗体（Inter Black），靠字距/字号/粗细营造高级感
- 按钮高度收窄：Medium 44px、Large 52px
- 主 Tab 改用纯文字下划线风格（Apple 风）
- 正文提升至 17px，行高 1.7-1.8

### iOS 26 质感升级
- 玻璃拟态轻量版：滚动导航栏、悬浮卡片、Hero 叠加元素
- 弹簧动效：按钮按下、卡片弹出使用 spring 弹性缓动
- 页面转场：淡入 + 轻微上浮（水平滑动仅用于抽屉/弹窗）
- 视差滚动：Hero 图片随滚动轻微视差
- 滚动吸附：横滑列表使用 scroll-snap
- 滚动渐显：板块元素进入视口时渐显上滑（stagger 错落 0.1s）
- 悬浮光晕：桌面端卡片 hover 时有轻微光晕跟随
- 微妙渐变：多色渐变（iOS 壁纸质感），不平涂
- 高光描边：按钮、卡片边缘极细高光，模拟玻璃/金属
- 噪点纹理：深色背景叠加极细微噪点，增加质感
- 深度感：模糊 + 半透明 + 微妙阴影营造层级深度

# 1. 设计系统 / Design System

## 1.1 色彩体系

> **核心原则：橙色 ≤ 5% 面积占比。大面积使用奶油白/纯白/深灰，橙色仅作点睛之笔。**

### 1.1.1 主色 Primary（橙）

| 色阶 | 色值 | 使用场景 |
|------|------|----------|
| Primary-50 | #FDF3EE | 极浅底，几乎不用 |
| Primary-100 | #F9E2D6 | 最浅底，极少使用 |
| Primary-300 | #EBA47E | 渐变最浅色 |
| **Primary-500** | **#C45A2C** | **核心强调色（品牌色，陶土橙）（CTA按钮、Logo、关键数据）** |
| Primary-600 | #A84A22 | 按钮悬停态 |
| Primary-700 | #8C3B1B | 按钮按下态 |

**主色使用原则（严格执行）：**
- ✅ 核心 CTA 按钮（每页 2-3 个）
- ✅ 品牌 Logo
- ✅ 关键数据强调（数字高亮）
- ✅ 价格标签（商城/课程）
- ❌ 输入框焦点态 → 用深灰描边
- ❌ Tab 选中态 → 用深灰文字 + 下划线
- ❌ 标签/徽章 → 用深灰底白字或浅灰底深灰字
- ❌ 评分星星 → 用深灰色
- ❌ 图标背景圆形 → 用 Gray-100 浅灰
- ❌ 筛选选中态 → 深灰文字 + 浅灰背景
- ❌ 大面积背景 → 绝不用

### 1.1.2 辅助色 Secondary

| 名称 | 色值 | 使用场景 |
|------|------|----------|
| 奶油白 | #FAF8F5 | 页面背景色，替代纯白，柔和高级 |
| 暖沙色 | #EDE6DE | 极小面积点缀（特色小卡片、单板块背景） |

> **注意**：已移除玫瑰金、鼠尾草绿。VIP/会员体系使用深灰 + 橙色点缀方案。

### 1.1.3 中性色 Neutral（核心色盘）

| Gray-200 | #E7E5E1 | 分割线、极细边框（默认卡片边框） |
| Gray-300 | #D1CFCA | 悬停态边框、输入框默认边框 |

## 1.5 阴影 Shadow（v2.0 大幅减少）

> **原则：平而不薄。默认无阴影，阴影仅用于真正"悬浮"的元素。**

| Token | CSS 值 | 使用场景 |
|-------|--------|----------|
| shadow-xs | `0 1px 2px rgba(0, 0, 0, 0.04)` | 极少使用 |
| **shadow-sm** | `0 2px 8px rgba(0, 0, 0, 0.06), 0 1px 2px rgba(0, 0, 0, 0.04)` | **下拉菜单、轻量浮层** |
| **shadow-md** | `0 8px 24px -4px rgba(0, 0, 0, 0.08), 0 2px 6px rgba(0, 0, 0, 0.04)` | **弹窗、悬浮卡片、导航吸顶** |
| shadow-lg | `0 20px 40px -8px rgba(0, 0, 0, 0.12), 0 4px 8px rgba(0, 0, 0, 0.04)` | 模态框、大型浮层 |
| shadow-nav | `0 1px 0 rgba(0, 0, 0, 0.06)` | 导航栏底部发丝线阴影 |
| shadow-orange | `0 8px 24px -4px rgba(196, 90, 44, 0.35)` | 主按钮悬停（仅此一处） |

**阴影使用原则（严格执行）：**
- ❌ 默认卡片 → 1px Gray-200 边框，无阴影
- ❌ 卡片悬停 → 不用阴影+上移，用 1px Gray-300 边框 + scale 1.01 极微放大
- ✅ 下拉菜单 → shadow-sm
- ✅ 弹窗/模态框 → shadow-md / shadow-lg
- ✅ 悬浮卡片（真正浮在内容上）→ shadow-md
- ✅ 吸顶导航 → shadow-nav（发丝线下沿）

## 1.9 玻璃拟态 Glass Morphism（v2.0 轻量版）

### 1.9.1 设计原则

Apple 风格轻薄通透，不是厚重毛玻璃：
- **轻薄感**：透明度高（82%-92% 白），模糊半径适中
- **层次分明**：配合内高光边 + 极细外描边
- **克制使用**：仅导航栏、悬浮卡片等浮层元素，不滥用
- **饱和度增强**：saturate(180%) 让背景颜色更鲜艳（iOS 风格）

### 1.9.2 玻璃面板等级

| 等级 | 背景 | 模糊 + 饱和度 | 内描边 | 外描边 | 使用场景 |
|------|------|-------------|--------|--------|----------|
| 导航级 | `rgba(255,255,255,0.82)` | `blur(20px) saturate(180%)` | `inset 0 -1px 0 rgba(0,0,0,0.06)` | - | 吸顶导航栏 |
| 悬浮级 | `rgba(255,255,255,0.88)` | `blur(16px) saturate(180%)` | `inset 0 1px 0 rgba(255,255,255,0.6)` | 1px rgba(255,255,255,0.3) | 悬浮卡片、下拉菜单 |
| 弹层级 | `rgba(255,255,255,0.92)` | `blur(24px) saturate(180%)` | `inset 0 1px 0 rgba(255,255,255,0.6)` | 1px rgba(0,0,0,0.06) | 模态框、弹窗 |

### 1.9.3 适用场景

1. **吸顶导航栏** — 滚动后玻璃态，内容从下方穿过
2. **悬浮数据卡** — Hero 区浮动信息卡
3. **下拉菜单 / 选择器** — 轻量玻璃感
4. **Hero 叠加标签** — 玻璃徽章/标签
5. **模态框 / 弹窗** — 标准玻璃面板

### 1.9.4 降级方案
- 不支持 backdrop-filter 时，使用半透明纯色 + 1px 边框
- 移动端模糊半径减半（12px）

# 4. 设计原则与交互规范

## 4.1 设计原则

### 4.1.1 核心原则

1. **女性友好 Female-First**
   - 从女性用户视角出发
   - 柔和温暖，但不甜腻
   - 图片真实自然，不贩卖身材焦虑

2. **极简克制 Minimal & Restrained**
   - 留白充足，信息层级清晰
   - 每页聚焦 1-2 个核心动作
   - 装饰元素极少，功能优先

3. **高级质感 Premium Feel**
   - 超大字号 + 巨量留白
   - 高质量摄影
   - 精致细节（圆角、微阴影、微动效）
   - **橙色 ≤ 5%**，克制的点缀才高级
   - **平而不薄**，不靠阴影堆层次，靠留白和排版

4. **信任感 Trust & Safety**
   - 信息透明
   - 反馈清晰
   - 隐私安全感

### 4.1.2 质感设计原则（v2.0 iOS 26 风格）

**Apple / Vision Pro 质感三要素：**

1. **通透感 Translucency**
   - 浮层用玻璃拟态，背景隐约可见
   - 轻薄通透，不是厚重磨砂
   - 仅限导航栏、浮层、弹窗

2. **层次感 Layering**
   - Z 轴层次清晰
   - 顶层：模态框（shadow-lg + 模糊）
   - 中层：下拉菜单、悬浮卡（shadow-sm + 模糊）
   - 底层：页面内容（无阴影，细边框）
   - 不靠阴影堆层次，靠模糊 + 半透明 + 位移

3. **生命力 Dynamism**
   - 即时柔和的动效反馈
   - 交错入场的节奏感
   - spring 弹性物理感
   - 视差滚动的空间感

### 4.1.3 少即是多（v2.0 核心思想）

- **颜色少**：橙、白、灰，三色为主
- **装饰少**：没有多余的图形元素
- **阴影少**：默认无阴影，悬浮才加
- **动效少**：微妙克制，不炫技
- **元素少**：每页信息密度低，留白充足
- **少，但是精** — 每个留下的元素都打磨到位

### 4.1.4 布局原则

- 内容居中对齐，视觉平衡
- 栅格严格对齐
- 阅读动线清晰
- 大板块间距（160-200px），呼吸感充足

## 4.2 动效规范 Motion（v2.0 iOS 26 风）

### 4.2.1 动效时长

| 类型 | 时长 | 场景 |
|------|------|------|
| 极快 | 100ms | 微交互、开关 |
| 快速 | 200ms | 按钮悬停、输入聚焦 |
| **标准** | **250ms** | **卡片悬浮、按钮回弹、小过渡** |
| 中速 | 350ms | 模态框、页面转场、抽屉 |
| 慢速 | 500ms | 大图渐显、元素入场 |
| 缓慢 | 800ms | 视差、装饰动画、Ken Burns |

> 核心区间 200-400ms，快速但柔和。

### 4.2.2 缓动函数

| 类型 | CSS 值 | 使用场景 |
|------|--------|----------|
| **标准缓动** | `cubic-bezier(0.25, 0.46, 0.45, 0.94)` | **大多数动效（v2.0 核心）** |
| 进入缓动 | `cubic-bezier(0, 0, 0.2, 1)` | 入场、渐显 |
| 退出缓动 | `cubic-bezier(0.4, 0, 1, 1)` | 退场、渐隐 |
| **弹性缓动** | `spring(1, 300, 20)` 或 `cubic-bezier(0.34, 1.56, 0.64, 1)` | **按钮按下、弹出、数字跳动（iOS 弹簧感）** |
| iOS 页面 | `cubic-bezier(0.32, 0.72, 0, 1)` | 页面切换、抽屉滑入 |

### 4.2.3 页面转场（v2.0 新增）

**1. 淡入上浮（主要页面转场）**
- 新页面：opacity 0 + translateY(12px) → opacity 1 + translateY(0)
- 旧页面：opacity 1 → opacity 0（交叉淡入）
- 时长：300ms
- 缓动：标准缓动
- 适用：所有页面跳转、列表→详情
- 注：网页端不使用水平滑动，避免与浏览器手势冲突

**2. Tab 切换淡入**
- 淡入 + 轻微上浮 4px
- 时长：250ms
- 适用：Tab 切换、同级内容切换

**3. 底部弹出（Bottom Sheet）**
- 底部滑入（translateY(100%) → 0）
- 遮罩淡入
- 时长：350ms
- 缓动：iOS 页面缓动
- 适用：移动端弹窗、操作菜单、侧边抽屉

### 4.2.4 元素入场（v2.0 核心）

**1. Staggered Fade-in-up（交错渐显上浮）**
- 初始：opacity 0 + translateY(24px)
- 结束：opacity 1 + translateY(0)
- 触发：滚动进入视口
- 时长：600ms
- 缓动：标准缓动
- 交错延迟：60-100ms/个
- 适用：卡片网格、列表、功能点
- **注意**：同一屏内交错不超过 5 个，太多反而杂乱

**2. 大标题显现**
- 整体或逐字渐显上移
- 时长：700ms
- 适用：Hero 标题、页面主标题

**3. 缩放出现**
- scale(0.94) + opacity 0 → scale(1) + opacity 1
- 弹性缓动
- 适用：模态框、徽章、弹出元素

**4. 数字滚动 Count-up（克制版）**
- 从 0 滚动到目标值
- 时长：800ms
- 缓动：ease-out（先快后慢，最后停稳）
- 不弹跳、无弹性、无千分位跳动
- 仅整数/一位小数变化
- 适用：关键数据展示（控制数量，一屏不超过 3 个）
- 备选：直接淡入出现（更极简的方案）

### 4.2.5 滚动视差（v2.0 新增）

**多层视差：**

| 层级 | 速度 | 元素 |
|------|------|------|
| 最远 | 0.3x | 背景渐变光、装饰 |
| 远层 | 0.5x | Hero 背景图 |
| 中层 | 0.8x | 浮动卡片、次要视觉 |
| 近层 | 1.0x | 主要文字内容 |
| 最近 | 1.2-1.5x | 前景特写元素（极少用） |

**实现：** CSS transform 或 JS 监听
**性能：** 移动端降级
**仅限：** Hero 区、大幅 Banner

### 4.2.6 滚动吸附 Scroll Snap（v2.0 新增）

- 横滑卡片列表使用 scroll-snap-type: x mandatory
- 吸附对齐：center
- 结合拖拽滑动
- 适用：评价轮播、课程横滑、商品横滑

### 4.2.7 悬浮光晕光标（v2.0 新增，桌面端）

- 卡片 hover 时，有极淡的橙色/白色光晕跟随鼠标
- 光晕：径向渐变，透明度 ≤ 10%
- 跟随延迟：100ms 缓动
- 效果：极微妙，增强交互反馈的高级感
- 仅限桌面端，移动端自动禁用

### 4.2.8 按钮反馈（v2.0 Spring 弹性）

- 悬停：颜色/背景变化 + 极微上移 1px
- 按下：scale(0.97)，100ms
- 释放：spring 弹性回到 scale(1)，带轻微过冲
- 类似 iOS 按钮的物理触感

### 4.2.9 Ken Burns 效果

- 图片缓慢放大（1.0 → 1.08）+ 微移
- 时长：15-20s 循环
- 缓动：ease-in-out
- 适用：Hero 背景、大图卡片
- 增加画面生命力，但非常克制

### 4.2.10 动效原则

- **克制**：不为动而动，服务于功能和体验
- **性能**：优先 transform + opacity，避免重排
- **尊重系统**：支持 prefers-reduced-motion
- **统一**：同类交互同模式
- **物理感**：spring 弹性、惯性缓动，自然真实

## 4.3 响应式断点

### 4.3.1 断点定义

| 断点 | 宽度范围 | 设备 |
|------|---------|------|
| xs | < 480px | 小屏手机 |
| sm | 480 - 767px | 手机 |
| md | 768 - 1023px | 平板 |
| lg | 1024 - 1439px | 笔记本 |
| xl | 1440 - 1919px | 桌面 |
| 2xl | ≥ 1920px | 大屏 |

### 4.3.2 栅格响应式

| 组件 | 移动端 | 平板 | 桌面 |
|------|--------|------|------|
| 课程卡片 | 1列 | 2列 | 3列 |
| 商品卡片 | 2列 | 3列 | 4列 |
| 教练卡片 | 1列 | 2列 | 4列 |
| 页脚 | 1列 | 2列 | 4列 |

### 4.3.3 导航响应式

- **桌面（≥1024px）**：完整水平导航
- **平板/移动（<1024px）**：汉堡菜单 + 侧边栏
- 无底部 Tab Bar

### 4.3.4 动效响应式

- **桌面**：完整动效（视差、玻璃模糊、staggered 入场）
- **平板**：简化（关闭视差，保留淡入）
- **移动端**：基础动效（淡入淡出、按钮反馈），模糊减半
- **低性能**：自动降级，仅 opacity 过渡

## 4.4 交互反馈规范

### 4.4.1 点击反馈

| 元素 | 悬停 | 按下 |
|------|------|------|
| 主按钮 | 提亮 + 橙影 + 微上移 | scale(0.97) + 变暗 |
| 次按钮 | 边框加深 + 浅灰底 | scale(0.98) |
| 卡片 | 边框加深 + scale(1.01) + 图片放大 | scale(0.995) |
| 文字链接 | 变深 + 下划线滑入 | 颜色更深 |
| 图标按钮 | Gray-100 背景 | 缩放 0.95 |

**过渡：** 200-250ms 标准缓动

### 4.4.2 加载状态

**按钮加载：** 内部旋转器，禁用

**页面加载：** 顶部进度条 2px（Gray-800）+ 骨架屏

**图片加载：**
- Gray-50 占位
- 400ms 淡入
- 可选：模糊到清晰（blur(16px) → 0）

### 4.4.3 空状态

- 大图标（40px），Gray-300
- 标题：16px，600，Gray-700
- 描述：14px，Gray-500
- 操作按钮：次按钮
- 居中
- 入场：缩放出现

### 4.4.4 错误处理

**表单错误：** 红边框 + 下方文字 + 轻微抖动
**操作失败：** Toast
**网络错误/404：** 居中插画 + 按钮

### 4.4.5 确认机制

- 删除、支付、重要变更需二次确认
- 移动端：底部 ActionSheet
- 桌面端：居中模态框

## 4.5 无障碍规范

### 4.5.1 色彩对比度
- 正文 ≥ 4.5:1
- 大标题 ≥ 3:1
- 不靠颜色单独传信息

### 4.5.2 键盘导航
- 全部可 Tab 聚焦
- 明显的聚焦指示器
- Tab 顺序与视觉一致

### 4.5.3 减少动效
- 尊重 prefers-reduced-motion
- 开启时关闭视差、3D、复杂动画
- 保留淡入淡出基本过渡

## 结语

> **v2.0 核心思想：克制即高级。**
>
> 橙色点缀而非铺陈，留白充足而非拥挤，微动效而非炫技，平而不薄而非层层堆叠。
> 像 Apple 官网那样 — 大量的留白、超大的标题、极精的细节、克制的动效。
>
> **品牌调性回顾：** 极简高级、生活方式感、专业精致。
> **核心价值回顾：** 安全、私密、无评判、高适配。
>
> 一切设计决策，回归这两句话。
